## Current thinking

We are exploring whether Sonicverse should package a small set of public primitives as
versioned modules instead of keeping them embedded in reference implementations.

## Questions

- What deserves to become a maintained package?
- Which pieces are only useful in the context of a full starter?
- How much public API surface can we support responsibly?
